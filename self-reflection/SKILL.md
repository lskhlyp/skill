# 反向自测技能 (self-reflection)

三阶段流程：**交互问答 → 自动沉淀 → 间隔回访**

## 触发条件

当用户输入以下任意指令时启动：
- `/self-reflection`

## 入口判断

技能启动后，**根据用户输入的第一判断**决定模式：

| 用户输入特征 | 模式 |
|-------------|------|
| 仅输入 `/self-reflection` ，**无后续具体话题/文件**（包括"随便"、"开始吧"等模糊指令） | → **模式A：AI出题自测**（AI随机选题，问2-3题，不限难度层级） |
| 输入中包含 **文件名**、文件路径，或"分析这个文件"、"帮我看看这个"等 | → **模式C：文档分析** |
| 用户表示**迷茫**、"不知道想干什么"、"没什么头绪"、"很乱"等状态 | → **模式B：自由对话（引导式）**  |
| 抛出具体 **观点/困惑/话题**（非文件，非模糊指令） | → **模式B：自由对话** |
| "考考我关于 X" / "出题测试 X"/ "问我几个 X" | → **模式A：AI出题自测**（按指定话题出题） |

---

## 模式A：AI出题自测

### Step A1: 确认话题

- **若用户指定了话题**（如"考考我关于 X"）→ 以此话题出题。
- **若用户未指定话题**（仅输入 `/self-reflection` ）→ **AI 自主选题**，从以下范畴中随机选一个，每次不重复：
  - **Wiki 概念页**（优先）：读取 `wiki/index.md`，从现有概念页（概念/实体/来源）中选题——优先选尚未被复盘过的概念
  - 人生哲学 / 价值观
  - 思维模型与认知偏误
  - 技术概念与原理
  - 历史与社科
  - 科学与方法论
  - 用户近期可能接触的领域（可从对话历史推断）
- **出题数量**：2-3 题，**不限制难度层级**，基础/应用/挑战随机混合。

### Step A2: AI 出题

根据上一步的选题逻辑出题：
- **有指定话题** → 按难度分层出题，至少 1 基础 + 1 应用/挑战
- **无指定话题（AI随机选题）** → 2-3 题，不限难度，基础/应用/挑战随机混合

难度分层参考：

**针对人生反思 / 哲学思辨类话题：**

| 层级 | 目的 | 示例 |
|------|------|------|
| 基础 | 检测核心观点是否理解到位 | "苏格拉底为什么说'无论怎么选都会后悔'？" |
| 应用 | 检测能否联系自身经历验证观点 | "你自己的生活中有哪个选择让你真正体会到了'遗憾是常态'？" |
| 挑战 | 检测能否突破原观点的框架做批判性思考 | "如果说遗憾是常态，那'不后悔地做选择'这件事本身是否就是一种需要练习的能力？" |

**针对技术 / 方法论类话题：**

| 层级 | 目的 | 示例 |
|------|------|------|
| 基础 | 检测基本概念是否清晰 | "什么是 Agent？" |
| 应用 | 检测能否解决实际问题 | "用 Agent 实现一个自动翻译工作流需要哪些组件？" |
| 挑战 | 检测深度思考能力 | "Agent 和传统状态机的本质区别是什么？何时不该用 Agent？" |

### Step A3: 批改反馈
用户作答后，AI 对其答案进行批改：
1. **事实性错误**：指出概念偏差、数据错误
2. **逻辑断层**：从 A 直接跳到 C，缺了 B
3. **认知盲区**：用户因经验限制忽略的视角
4. **表达清晰度**：是否真正理解了，还是只是在复述术语

允许用户追问，直到真正理解。

## 模式B：自由对话

用户直接抛出问题、观点、或话题，AI 以苏格拉底式对话纵深挖掘。

### Step B1: 接收输入
直接接收用户的提问/观点/话题。

### Step B2: 苏格拉底式纵深

**你现在是顶尖的思维教练。** 不要急着给答案，而是通过对话帮助用户把想法想透：

1. **澄清** — "你说的 X，具体是指什么？能举个例子吗？"
2. **反诘** — "有没有相反的情况？如果 X 成立，那 Y 为什么也会发生？"
3. **类比** — "这跟 Z 是不是类似？它们本质上的区别在哪？"
4. **推演** — "如果你的这个前提成立，推导下去会得到什么结论？"
5. **溯源** — "你为什么会有这个想法？是来自某本书、某个人、还是自己的经历？"

每轮追问一层，直到触及用户的知识边界或思维盲区。

### Step B3: 对话总结
当用户表示"差不多了"或对话已触及足够深度时，提炼为结构化复盘：

1. **【核心话题】**：用户最初提出的问题/观点
2. **【对话脉络】**：从起点到终点的思维路径摘要
3. **【浮现的盲区】**：对话中暴露出的思维短板、知识缺口
4. **【关键记忆点】**：2-3 个最值得记住的洞察
5. **【行动建议】**：下一步可以读什么、想什么、做什么

## 模式C：文档分析

对用户提供的文件进行深度分析。

### Step C1: 确认目标文件
1. 如果用户直接提供了文件名，确认文件存在。
2. 如果用户只说了"分析一个文件"，列出当前目录文件让用户选择。

### Step C2: 读取文件
使用 `Read` 工具读取文件内容。

### Step C2.5: Q&A 检测
读取后判断内容是否包含**明确的问题-回答对**格式（如 `Q:... A:...`、`问：... 答：...`、`问题 N：... 回答：...`、`Q&A` 标题等）：
- **是** → 跳过 Step C3，直接进入[[#统一收尾（三种模式共用）]]。复盘将评估每道回答的正确性，提取知识漏洞。
- **否** → 继续 Step C3 深度分析。

### Step C3: 深度分析

**你现在是顶尖的思维教练和学科专家。**

#### A. 知识点核查 (Knowledge Check) — 基础层
- 是否存在**事实性错误**或**定义偏差**？
- 是否存在**关键概念缺失**（只讲了A没讲B，逻辑不闭环）？

#### B. 应用层验证 (Application Validation)
- 概念能否真正用于解决实际问题？理论与实践之间有无差距？
- 是否有"我以为懂了，但实际上经不起推敲"的地方？

#### C. 思维模式拆解 (Thinking Pattern) — 挑战层
- 逻辑链条是否存在**跳跃**？
- 是否陷入**循环论证**或**幸存者偏差**？
- 因立场或经验限制忽略了哪些**盲区**？

#### D. 结构与表达 (Structure)
- 信息组织是否清晰？有无**冗余**或**混乱**？

## 统一收尾 (三种模式共用)

无论哪种模式，完成后都走以下流程

### 生成复盘文件

**文件命名规则：**
- 模式A：`AI出题-[话题简述]-复盘-[YYYYMMDD].md`
- 模式B：`自由对话-复盘-[YYYYMMDD].md`
- 模式C（文档分析）：`[原文件名]-复盘-[YYYYMMDD].md`
- 模式C（Q&A检测）：`[原文件名]-复盘-[YYYYMMDD].md`（同模式C命名，frontmatter 标记 `qa-review`）

**文件 frontmatter：**
```yaml
---
type: review
tags: [self-test, 话题标签]
mode: ai-quiz | free-chat | document | qa-review
wiki_ref: [关联的概念页名]       # 关联的 wiki 概念页（可选）
sources:
  - wiki/概念/关联概念页         # 关联来源（可选）
review_round: N
intervals:
  next_1d: YYYY-MM-DD
  next_7d: YYYY-MM-DD
  next_30d: YYYY-MM-DD
created: YYYY-MM-DD
updated: YYYY-MM-DD
---
```

**内容结构：**
1. **【时间/地点】**：YYYY-MM-DD 星期X | xx省 xx市
2. **【核心诊断】**：一句话总结最大问题（模式C）、核心话题（模式A/B）、或 Q&A 回答质量总体评估（qa-review 模式）
3. **【知识漏洞清单】**（标记难度层级）或 **【浮现的盲区】** — qa-review 模式下从错误/不准确的回答中提取
4. **【思维短板分析】** 或 **【对话脉络】** — qa-review 模式下分析答错原因（概念不清/理解偏差/遗漏要点等）
5. **【关键记忆点】**：2-3 个最核心、最易忘的点（用于间隔回访）
6. **【行动建议】**：
   - 下一步查阅什么资料或如何修改
   - 如果漏洞涉及的概念尚未在 wiki 中建立页面 → 触发创建
7. **【关联 Wiki 页面】**：通过 wiki-link 引用本次涉及的所有概念/实体/来源页，如 `[[元素异常]]`

### 更新漏洞追踪

检查 `wiki/复盘/漏洞待办.md` 文件：
- **不存在**：创建一个
- **已存在**：读取后追加新条目，标记重复

```markdown
# 知识漏洞追踪

| 日期 | 漏洞描述 | 来源 | 模式 | 难度 | 状态 |
|------|---------|------|------|------|------|
| YYYY-MM-DD | 不清楚 X 概念 | xxx.md | 出题/自由/文档 | 基础/应用/挑战 | ❌ 待补充 |
```

状态说明：
- `❌ 待补充` — 尚未查找资料填补
- `🔄 学习中` — 正在补充
- `✅ 已补充` — 已学习，等待间隔回访验证
- `✔️ 已验证` — 间隔回访通过，确认掌握

### 间隔回访安排

每次生成复盘文件时，一并输出**间隔回访安排表**展示给用户：

```
📅 间隔回访安排
  ├ 1天后  (MM-DD) → 快速测验关键记忆点
  ├ 7天后  (MM-DD) → 纵深追问，检查是否内化
  └ 30天后 (MM-DD) → 全面复盘，对比上次有无进步
```

**当用户说"来回访"或时间到达回访日时：**
1. 读取对应的复盘文件，提取 `key_memory_points` 或 **【关键记忆点】**
2. 只针对历史漏洞点提问（不引入新话题）
3. 用户回答后判断漏洞是否已解决
4. 更新 `wiki/复盘/漏洞待办.md` 中对应条目的状态

### 输出与反馈
1. 使用 `Write` 工具将复盘文件写入 `wiki/复盘/` 目录。
2. 更新（或创建）`wiki/复盘/漏洞待办.md`。
3. **同步 wiki 索引与日志：**
   - 追加 `wiki/log.md`：`## [YYYY-MM-DD] review | 话题简述`
   - 追加 `wiki/index.md` 中"复盘记录"表格行，更新 Stats 总页面数 +1
4. 向用户展示写入成功的消息，附上间隔回访安排表。
5. 如果涉及 wiki 中尚未建立的概念且已在 ≥2 篇来源中出现，提示用户启动 Ingest 工作流。

## 工具使用规范
- **Bash**: 列出文件、确认路径。
- **Read**: 读取待分析文件、已有的 `wiki/复盘/漏洞待办.md`、历史复盘文件。
- **Write**: 创建复盘文件到 `wiki/复盘/`、更新 `wiki/复盘/漏洞待办.md`、更新 `wiki/index.md` 和 `wiki/log.md`。
- **Glob/Grep**: 检查 wiki 中是否已存在相关概念/实体页面。出题前读取 `wiki/index.md` 获取可用概念列表。
